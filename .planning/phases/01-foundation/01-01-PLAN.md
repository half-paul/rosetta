1. Create `src/db/schema.ts` with ALL tables. Per D-08 (CUID2 PKs): use `text('id').primaryKey().$defaultFn(() => createId())` from `@paralleldrive/cuid2` for domain table PKs instead of `uuid().defaultRandom()`. Install cuid2: `pnpm add @paralleldrive/cuid2`.

   NOTE: D-08 locks CUID2. RESEARCH.md patterns show uuid -- using CUID2 per user decision D-08.

   Auth tables (required by @auth/drizzle-adapter -- keep column names EXACTLY as shown, per Pitfall 3):
   ```typescript
   // users table: keep all NextAuth required columns unchanged.
   // Add alongside (not rename): passwordHash, role, createdAt, updatedAt, deletedAt
   export const users = pgTable('user', {
     id: text('id').primaryKey().$defaultFn(() => crypto.randomUUID()),  // NextAuth requires text id
     name: text('name'),
     email: text('email').unique(),
     emailVerified: timestamp('emailVerified', { mode: 'date' }),
     image: text('image'),
     // Application extensions (Pitfall 6: passwordHash lives here):
     passwordHash: text('password_hash'),
     role: text('role').notNull().default('reviewer'),
     createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
     updatedAt: timestamp('updated_at', { withTimezone: true }).notNull().defaultNow(),
     deletedAt: timestamp('deleted_at', { withTimezone: true }),
   })
   ```

   accounts, sessions, verificationTokens: exactly as shown in PATTERNS.md.

   Domain tables (all use CUID2 text PKs per D-08, soft delete per D-09, timestamps per D-10):
   - `reviewStatusEnum`: pgEnum with values ['PENDING', 'AI_ANALYZED', 'HUMAN_APPROVED', 'HUMAN_REJECTED', 'PUBLISHED']
   - `articles`: id (cuid2), title, wikiUrl (unique), revisionId, language (default 'en'), fetchedAt, createdAt, updatedAt, deletedAt
   - `sections`: id (cuid2), articleId (FK->articles cascade), title, path, position, createdAt, updatedAt, deletedAt
   - `paragraphs`: id (cuid2), sectionId (FK->sections cascade), stableId (text, not null -- section_path:content_hash:revision_id), content, contentHash, position, createdAt, updatedAt, deletedAt
   - `claims`: id (cuid2), paragraphId (FK->paragraphs cascade), text, severity (text, nullable -- for Phase 3), createdAt, updatedAt, deletedAt
   - `commentaries`: id (cuid2), claimId (FK->claims cascade), draftText, status (reviewStatusEnum default 'PENDING'), createdAt, updatedAt, deletedAt
   - `reviews`: id (cuid2), commentaryId (FK->commentaries cascade), reviewerId (FK->users), previousStatus (reviewStatusEnum), newStatus (reviewStatusEnum), editedText (nullable), reviewedAt, createdAt
   - `scores`: id (cuid2), articleId (FK->articles cascade, unique), factualScore (integer default 0), coveragePercent (integer default 0), totalParagraphs (integer default 0), reviewedParagraphs (integer default 0), updatedAt, createdAt

   Import cuid2:
   ```typescript
   import { createId } from '@paralleldrive/cuid2'
   ```

   For domain table PKs:
   ```typescript
   id: text('id').primaryKey().$defaultFn(() => createId()),
   ```

2. Create `src/db/index.ts`:
   ```typescript
   import 'server-only'
   import { Pool } from 'pg'
   import { drizzle } from 'drizzle-orm/node-postgres'
   import * as schema from './schema'

   if (!process.env.DATABASE_URL) {
     throw new Error('DATABASE_URL is required')
   }

   const pool = new Pool({ connectionString: process.env.DATABASE_URL })
   export const db = drizzle(pool, { schema })
   ```
   NOTE: Using standard `pg` Pool for local Docker development per D-01. Neon serverless driver is for future Vercel deployment (D-03 deferred).

3. Create `src/types/index.ts`:
   ```typescript
   import type {
     articles, sections, paragraphs, claims, commentaries, reviews, scores, users
   } from '@/db/schema'

   export type Article = typeof articles.$inferSelect
   export type NewArticle = typeof articles.$inferInsert
   export type Section = typeof sections.$inferSelect
   export type NewSection = typeof sections.$inferInsert
   export type Paragraph = typeof paragraphs.$inferSelect
   export type Claim = typeof claims.$inferSelect
   export type Commentary = typeof commentaries.$inferSelect
   export type Review = typeof reviews.$inferSelect
   export type Score = typeof scores.$inferSelect

     | 'PENDING'
     | 'AI_ANALYZED'
     | 'HUMAN_APPROVED'
     | 'HUMAN_REJECTED'
   ```
