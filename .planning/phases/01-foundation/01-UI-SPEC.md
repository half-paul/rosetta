# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

Phase 1 delivers exactly two UI surfaces:

1. **Login page** (`/login`) — email/password form + OAuth buttons (Google, GitHub)
2. **Dashboard shell** (`/dashboard`) — protected skeleton; no data content (populated in Phase 4)

All other UI work (article queue, reviewer workflow, public pages) is explicitly out of scope for this phase.

---

| Property | Value | Source |


## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline badge padding |
| sm | 8px | Input internal padding (vertical), button icon gap |
| md | 16px | Form field spacing, card internal padding |
| lg | 24px | Card section gaps, between form groups |
| xl | 32px | Card padding (desktop), page vertical centering offset |
| 2xl | 48px | Login card max-width inner breathing room |
| 3xl | 64px | Page-level vertical centering (login page flex gap) |

Exceptions:
- Touch targets for OAuth buttons: minimum 44px height (accessibility requirement — WCAG 2.5.5)
- Login card: `max-w-sm` (384px) centered on page

|------|------|--------|-------------|-------|

| Label | 14px | 600 (semibold) | 1.4 | Form field labels, button text, nav items |

Monospace (for IDs, hashes if shown): `var(--font-geist-mono)` — loaded alongside Geist Sans.

## Color

shadcn default palette (CSS variables on `:root`). No custom overrides in Phase 1.

| Role | CSS Variable | Light Value | Dark Value | Usage |
|------|-------------|-------------|------------|-------|
| Dominant (60%) | `--background` | `#ffffff` | `#09090b` | Page background, login page surface |
| Secondary (30%) | `--card` / `--muted` | `#f4f4f5` | `#18181b` | Login card background, dashboard nav, form areas |
| Accent (10%) | `--primary` | `#18181b` | `#fafafa` | Sign in button, focused input ring, active nav item |
| Destructive | `--destructive` | `#ef4444` | `#7f1d1d` | Auth error alert border, future destructive actions |

Accent reserved for:
- Primary "Sign in" button fill
- Input focus ring (`ring-2 ring-primary`)
- Active/current nav item indicator in dashboard shell
- Link text in error alerts ("contact your administrator")

Color mode: Support both light and dark via shadcn CSS variables. Do NOT hardcode hex values — always use `bg-background`, `text-foreground`, `bg-card` etc. Tailwind v4 resolves these automatically.

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Login page title | "Welcome back" | Centered above card |
| Login page subtitle | "Sign in to your Rosetta account" | Muted text below title |
| Email field label | "Email address" | |
| Password field label | "Password" | |
| Primary CTA | "Sign in" | Submit button — verb only, no noun needed |
| OAuth divider | "or continue with" | Lowercase, muted color, flanked by `<Separator>` |
| Google OAuth button | "Continue with Google" | Google logo icon + text |
| GitHub OAuth button | "Continue with GitHub" | GitHub logo icon + text |
| Empty state heading (dashboard) | "No articles yet" | Centered in dashboard main area |
| Empty state body (dashboard) | "Articles you submit for fact-checking will appear here. Come back after Phase 2 is complete." | Muted body text below heading |
| Auth error — wrong credentials | "Incorrect email or password. Try again or contact your administrator." | Shown in `<Alert variant="destructive">` above form |
| Auth error — account not found | "No account found for this email. Rosetta uses invite-only access — contact your administrator." | Same alert component |
| Auth error — OAuth failure | "Sign-in failed. Try again or use email and password." | Same alert component |

| Loading state (sign in button) | "Signing in..." | Button text while submitting; button disabled |
| Loading state (OAuth button) | Spinner only, button disabled | No text change on OAuth buttons |

Destructive actions in Phase 1: None. Invite-only access means no account creation or deletion UI in this phase.

## Interaction Contract

### Login Form Behavior

| State | Behavior |
|-------|----------|
| Default | Email input focused on page load (`autoFocus`) |
| Typing | No inline validation — validate only on submit |
| Submitting | "Sign in" button: disabled + text → "Signing in..." + spinner |
| Error | `<Alert>` appears above form; inputs remain filled; focus returns to email field |
| Success | Redirect to `/dashboard` (handled by NextAuth `callbackUrl`) |

### OAuth Button Behavior

| State | Behavior |
|-------|----------|
| Default | Full-width button, provider logo left-aligned |
| Clicked | Button disabled + spinner (no text change) |
| Error | Auth error alert appears above form |
| Success | Redirect to `/dashboard` |

### Dashboard Shell Behavior

| State | Behavior |
|-------|----------|
| Authenticated | Renders nav shell + empty state in main area |
| Unauthenticated | Middleware redirects to `/login` before page renders |
| Loading | Next.js App Router Suspense boundary; nav skeleton visible immediately |

### Protected Route

Middleware (`middleware.ts`) intercepts all `/dashboard/*` routes. Unauthenticated requests receive a 307 redirect to `/login?callbackUrl=/dashboard`. No flash of protected content.

## Layout Contract

### Login Page (`/login`)

```
┌─────────────────────────────────────┐
│         (page bg: --background)     │
│                                     │
│   ┌─────────────────────────────┐   │
│   │       Welcome back          │   │  ← heading (24px, semibold)
│   │  Sign in to your Rosetta    │   │  ← subtitle (14px, muted)
│   │         account             │   │
│   │                             │   │
│   │  [Email address          ]  │   │  ← Input
│   │  [Password               ]  │   │  ← Input
│   │                             │   │
│   │  [      Sign in          ]  │   │  ← Button (primary, full-width)
│   │                             │   │
│   │  ─────  or continue with ─────  │  ← Separator + muted text
│   │                             │   │
│   │  [G]  Continue with Google  │   │  ← Button (outline, full-width)
│   │  [GH] Continue with GitHub  │   │  ← Button (outline, full-width)
│   └─────────────────────────────┘   │
│                                     │
└─────────────────────────────────────┘
```

- Page: `min-h-screen flex items-center justify-center bg-background`
- Card: `w-full max-w-sm` with `p-8` (32px internal padding)
- Title + subtitle: above card border, not inside card (floating header pattern)
- Error alert: inside card, above email field, `mb-4`

### Dashboard Shell (`/dashboard`)

```
┌─────────────────────────────────────────┐
│  [Rosetta]          [Reviewer name ↓]   │  ← Topnav (h-14, border-b)
├─────────────────────────────────────────┤
│                                         │
│         No articles yet                 │  ← Empty state heading (24px, semibold)
│   Articles you submit for fact-checking │  ← Empty state body (14px, muted)
│   will appear here. Come back after...  │
│                                         │
└─────────────────────────────────────────┘
```

- Layout: `flex flex-col min-h-screen`
- Topnav: `h-14 border-b bg-background px-6 flex items-center justify-between`
- Main: `flex-1 flex items-center justify-center` (centers empty state)
- No sidebar in Phase 1 — sidebar added in Phase 4 when queue exists

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Form labels | `<Label htmlFor="email">` and `<Label htmlFor="password">` — explicit `for` association |
| Error announcement | `<Alert role="alert">` — screen readers announce on insertion |
| Button loading state | `aria-disabled="true"` + `aria-busy="true"` on submit button while loading |
| OAuth icons | Icon SVGs have `aria-hidden="true"`; button text provides label |
| Focus order | Email → Password → Sign in → Google → GitHub (tab order matches visual order) |
| Color contrast | shadcn default tokens meet WCAG AA (4.5:1 for normal text, 3:1 for large) |
| Minimum touch target | OAuth buttons: `min-h-[44px]` |
