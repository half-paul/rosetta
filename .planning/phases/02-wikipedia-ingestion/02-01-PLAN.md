       ```typescript
       export async function mediawikiFetchWithBackoff(
         url: string,
         init?: RequestInit,
         maxRetries = 5,
       ): Promise<Response> {
         let delay = 1_000  // 1s initial per D-15
         for (let attempt = 0; attempt <= maxRetries; attempt++) {
           const res = await mediawikiFetch(url, init)
           if (res.status !== 429) return res
           if (attempt === maxRetries) {
             throw new Error(`MediaWiki rate limited after ${maxRetries} retries`)
           }
           await new Promise(r => setTimeout(r, delay))
           delay = Math.min(delay * 2, 32_000)  // cap at 32s per D-15
         }
         throw new Error('unreachable')
       }
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> URL normalization | Untrusted URL string from request body crosses into server-side processing |
| Server -> MediaWiki API | Outbound HTTP request to external service with user-influenced parameters |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | normalizeWikipediaUrl | mitigate | Validate hostname is exactly `en.wikipedia.org` or `en.m.wikipedia.org`; reject all other hostnames. Extract title only from `/wiki/` path prefix. |
| T-02-02 | Tampering | fetchArticle | mitigate | Build URL server-side using `URL` constructor with `searchParams.set()` — user-supplied title is a single URL parameter, never interpolated into path or query raw. Parse response as JSON, never eval. |
| T-02-03 | DoS | mediawikiFetchWithBackoff | accept | Backoff caps at 32s with max 5 retries. Individual job timeout handled by pg-boss expireInSeconds. No amplification vector — 1 user request = 1 API call chain. |
</threat_model>
