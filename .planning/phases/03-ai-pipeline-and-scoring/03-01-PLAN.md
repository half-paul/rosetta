
No mocks needed — this is a pure function. Write all 7 test cases from the behavior block above.

Key formulas for expected values:
- Coverage = (reviewed / total) * 100
- Accuracy = mean of accuracy ratings * 100, or 0 if empty array
- Confidence = mean of all confidence scores * 100, or 0 if empty
- Score = round(coverage * weights.coverage + accuracy * weights.accuracy + confidence * weights.confidence)
- Clamped to [0, 100]

For Test 1 (all unreviewed, confidence 0.8): coverage=0, accuracy=0, confidence=80 -> score = 0*0.4 + 0*0.4 + 80*0.2 = 16
For Test 2 (all reviewed, accuracy 1.0, confidence 1.0): coverage=100, accuracy=100, confidence=100 -> score = 40+40+20 = 100

**GREEN phase — implement** in `src/features/analysis/score-engine.ts`:

Create the file with `import 'server-only'` (since `computeAndPersistScore` touches DB).

Export `ScoreWeights` interface with `coverage`, `accuracy`, `confidence` fields (all `number`).
Export `DEFAULT_WEIGHTS: ScoreWeights = { coverage: 0.4, accuracy: 0.4, confidence: 0.2 }`.

Export `computeFactualScore(paragraphs, accuracyRatings, weights)` — pure function, returns `{ factualScore, coveragePercent, coverageComponent, accuracyComponent, confidenceComponent }`.

Handle edge case: if `paragraphs.length === 0`, return `{ factualScore: 0, coveragePercent: 0, coverageComponent: 0, accuracyComponent: 0, confidenceComponent: 0 }`.

Export async `computeAndPersistScore(articleId: string)`:
- Query all paragraphs for the article (join through sections)
- Query all claims for confidence scores
- Query all reviews for accuracy ratings (empty in Phase 3 — defaults to 0)
- Call `computeFactualScore()`
- Upsert into `scores` table using `db.insert(scores).values(...).onConflictDoUpdate({ target: scores.articleId, set: { ... } })`
- Include all component columns: `coverageComponent`, `accuracyComponent`, `confidenceComponent`, `scoreWeightsConfig`

The `computeAndPersistScore` function imports `db` from `@/db`, `scores`, `claims`, `paragraphs`, `sections` from `@/db/schema`, and `eq` from `drizzle-orm`.

For the paragraph query to determine `isReviewed`: a paragraph is considered reviewed if ALL of its claims' commentaries have status `HUMAN_APPROVED` or `PUBLISHED`. In Phase 3, no paragraphs will be reviewed (all commentary starts as PENDING/AI_ANALYZED), so coverage will always be 0%. The function should query commentaries status and aggregate per paragraph.

Simpler Phase 3 approach: since no human reviews exist yet, `isReviewed` can be determined by checking if any commentary on the paragraph's claims has status `HUMAN_APPROVED` or `PUBLISHED`. If none do, `isReviewed = false`. This query pattern will work correctly when Phase 4 adds reviews.
