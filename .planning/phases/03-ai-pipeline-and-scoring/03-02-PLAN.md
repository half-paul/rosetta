Create `src/features/analysis/claim-extractor.ts`:

```
import 'server-only'
import { generateText, Output } from 'ai'
import { registry } from '@/lib/ai-registry'
import { ClaimExtractionSchema, type ClaimExtractionOutput } from './schemas'
```

Define a `CLAIM_EXTRACTION_SYSTEM_PROMPT` constant string. The prompt must:
- Instruct the model to extract check-worthy factual claims from the given paragraph
- Define check-worthy per D-02: factual assertions that can be independently verified — exclude opinions, definitions, tautologies, trivially true statements
- Instruct severity assignment per D-04: "high" for health/safety/legal claims, "medium" for historical/scientific facts, "low" for trivial assertions
- Instruct that character offsets must point to the exact span of the claim text within the paragraph
- Instruct that confidence (0-1) reflects how certain the model is that this is a check-worthy claim
- Instruct to return an empty claims array if no check-worthy claims exist (D-03)
- CRITICAL per security threat T-03-01: The system prompt MUST separate instructions from content. The paragraph text goes in the `prompt` parameter, not concatenated into the system prompt.

Export `async function extractClaims(paragraphText: string): Promise<ClaimExtractionOutput>`:
- Call `generateText` with `model: registry.languageModel(process.env.AI_MODEL!)`, `output: Output.object({ schema: ClaimExtractionSchema })`, `system: CLAIM_EXTRACTION_SYSTEM_PROMPT`, `prompt: paragraphText`
- Return `output` (NOT `result.object` — that is the deprecated v5 shape)
- Wrap in try/catch: on any error, log `console.error('extractClaims failed, returning empty:', err)` and return `{ claims: [] }`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM output -> application | Untrusted AI-generated content validated by Zod schema before use |
| Wikipedia paragraph -> LLM prompt | Article content is untrusted input; must be isolated from system instructions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | claim-extractor.ts, commentary-drafter.ts | mitigate | System prompt in `system:` parameter, paragraph content in `prompt:` parameter — never concatenated. Prevents prompt injection via Wikipedia content. |
| T-03-03 | Information Disclosure | commentary-drafter.ts | mitigate | `isVerified: z.literal(false)` in schema + fallback always sets `isVerified: false`. No code path produces a verified AI source. |
| T-03-04 | Information Disclosure | claim-extractor.ts, commentary-drafter.ts | mitigate | `import 'server-only'` prevents client-side bundling of LLM API key access |
</threat_model>
