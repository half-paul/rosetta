**1. Create `src/features/analysis/analysis-worker.ts`:**

```
import 'server-only'
import { extractClaims } from './claim-extractor'
import { draftCommentary } from './commentary-drafter'
```

Export `async function runAnalysisJob(data: { articleId: string }): Promise<void>`:

Per D-18: single per-article job that iterates all paragraphs.

Step 1: Query all paragraphs for the article. Join through sections:
```
const articleParagraphs = await db.select({
  id: paragraphs.id,
  content: paragraphs.content,
  sectionId: paragraphs.sectionId,
}).from(paragraphs)
  .innerJoin(sections, eq(paragraphs.sectionId, sections.id))
  .where(eq(sections.articleId, data.articleId))
```

Step 2: Iterate each paragraph. Per RESEARCH Pitfall 4, commit per-paragraph (no wrapping transaction). For each paragraph:

a) Call `extractClaims(paragraph.content)` — returns `{ claims: [...] }` or `{ claims: [] }`

b) For each extracted claim, insert into `claims` table:
```
const [savedClaim] = await db.insert(claims).values({
  paragraphId: paragraph.id,
  text: claim.text,
  severity: claim.severity,
  confidenceScore: claim.confidenceScore,
  charOffsetStart: claim.charOffsetStart,
  charOffsetEnd: claim.charOffsetEnd,
}).returning()
```

c) For each saved claim, call `draftCommentary(paragraph.content, { text: claim.text, severity: claim.severity })`. Insert the result into `commentaries`:
```
await db.insert(commentaries).values({
  claimId: savedClaim.id,
  draftText: commentary.analysisText,
  status: 'AI_ANALYZED',    // per D-08
  suggestedSources: commentary.suggestedSources,
})
```

Note: `suggestedSources` is JSONB — Drizzle handles the JSON serialization automatically when the column is typed as `jsonb`.

d) Per D-03: If `extractedClaims.claims.length === 0`, the paragraph still counts as analyzed. No claims to insert, but the paragraph is processed. (Score engine counts all paragraphs regardless of claim count for coverage.)

e) Wrap each paragraph iteration in try/catch. On error, log `console.error('Analysis failed for paragraph:', paragraph.id, err)` and continue to the next paragraph (per RESEARCH Pitfall 4 — per-paragraph error handling).

f) Per D-21: After each paragraph, apply rate limiting delay:
```
await new Promise(resolve => setTimeout(resolve, Number(process.env.ANALYSIS_RATE_LIMIT_MS ?? '500')))
```

Step 3: After all paragraphs processed, call `await computeAndPersistScore(data.articleId)`.

Step 4: Log completion: `console.log('Analysis complete for article:', data.articleId)`

|----------|-------------|
| LLM output -> DB persistence | AI-generated claims and commentary enter the database; validated by Zod before insertion |
| Ingestion -> Analysis queue | Analysis jobs only created by authenticated ingestion path, not public API |
| Score engine -> DB | Score upsert only called from analysis worker or future review handler |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | analysis-worker.ts | mitigate | Per-paragraph commit prevents a single LLM failure from corrupting entire article analysis; Zod validates all LLM output before DB insert |
| T-03-06 | Denial of Service | analysis-worker.ts | mitigate | Rate limiting delay (D-21) between paragraphs; pg-boss retry limit (3) + dead-letter prevents infinite retry; job expiry at 15 min |

</threat_model>
