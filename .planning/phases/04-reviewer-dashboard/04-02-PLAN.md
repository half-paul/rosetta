<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> API routes | All reviewer actions cross this boundary; untrusted input arrives |
| API -> state machine | Status transitions must be validated server-side, not trusted from client |
| API -> audit_log | Every action must produce an immutable audit trail |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering | approve route | mitigate | API validates all sources rated (rating !== null) AND calls validateTransition() before any DB write; client disabled button is UX only |
| T-04-05 | Tampering | all routes | mitigate | Every status-changing route calls validateTransition(); direct API calls cannot bypass state machine |
| T-04-06 | Spoofing | all routes | mitigate | auth() check with session.user.id validation on every route; return 401 if missing |
| T-04-07 | Tampering | verify-source | mitigate | SELECT FOR UPDATE in transaction prevents concurrent JSONB race conditions |
| T-04-08 | Information Disclosure | error handling | mitigate | Generic 500 error response; err.message never exposed in HTTP body |
| T-04-09 | Elevation of Privilege | flag route | accept | Any authenticated reviewer can flag content; no role escalation — all users are reviewers in v1 |
</threat_model>
