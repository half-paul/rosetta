## Spacing Scale

Standard 8-point scale. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, tight inline spacing |
| sm | 8px | Compact element spacing, list item vertical padding |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, pane header padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Split pane top offset, page-level vertical rhythm |
| 3xl | 64px | Reserved — not used in this phase |

Exceptions:
- Topnav height: 56px (h-14 / 3.5rem) — established by Phase 1 dashboard shell, locked
- Touch targets for icon-only buttons: 44px minimum height (keyboard shortcut buttons)
- Split pane: left pane 50% width, right pane 50% width, divider 1px border

## Typography

All type uses `--font-sans` (Geist Sans). Mono font (`--font-mono`) reserved for code/technical snippets only (not used in this phase).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Queue item secondary text, source URL, muted metadata |
| Label | 14px (text-sm) | 600 (semibold) | 1.4 | Queue item article title, source card title, form labels |
| Heading | 16px (text-base) | 600 (semibold) | 1.4 | Section headings within pane, claim heading |

Source: Body and Label sizes inferred from dashboard shell (`text-sm`, `text-base`); weights from shadcn base-nova convention (400 + 600 only).

## Color

Color tokens are OKLCH CSS variables defined in `src/app/globals.css`. Use semantic token names — never raw hex values in component code.

| Role | Token | Light mode approx. | Dark mode approx. | Usage |
|------|-------|--------------------|--------------------|-------|
| Dominant (60%) | `--background` | white | near-black | Page background, topnav background |
| Secondary (30%) | `--card` / `--muted` | off-white | dark gray | Queue item cards, source cards, review pane backgrounds |
| Accent (10%) | `--primary` | near-black | near-white | See reserved list below |
| Sidebar surface | `--sidebar` | light gray | dark gray | Left navigation / sidebar (if added) |
| Destructive | `--destructive` | red (oklch 0.577 0.245 27.325) | red (oklch 0.704 0.191 22.216) | Reject button, HUMAN_REJECTED status badge |
| Muted text | `--muted-foreground` | medium gray | medium gray | Secondary metadata, timestamps, empty state body |
| Border | `--border` | light gray | 10% white | Card borders, pane divider, separator |

Accent (`--primary`) reserved exclusively for:
1. The primary approve CTA button (`Button` variant `default`)
2. Active/selected queue item highlight (left border accent)
3. "Flag" button that appears on text selection
4. Source verification progress counter text when all sources are verified

Traffic light source ratings use semantic colors outside the 60/30/10 palette — these are functional indicators, not brand accent:
- Confirms: `text-green-600` / `bg-green-50` (dark: `text-green-400` / `bg-green-950`)
- Partially supports: `text-amber-600` / `bg-amber-50` (dark: `text-amber-400` / `bg-amber-950`)
- Does not support: `text-red-600` / `bg-red-50` (dark: `text-red-400` / `bg-red-950`)

Severity badge colors:
- HIGH: `bg-red-100 text-red-700` (dark: `bg-red-950 text-red-300`)
- MED: `bg-amber-100 text-amber-700` (dark: `bg-amber-950 text-amber-300`)
- LOW: `bg-muted text-muted-foreground`

Status badge colors:
- PENDING: `bg-muted text-muted-foreground`
- AI_ANALYZED: `bg-blue-100 text-blue-700` (dark: `bg-blue-950 text-blue-300`)
- HUMAN_APPROVED: `bg-green-100 text-green-700` (dark: `bg-green-950 text-green-300`)
- HUMAN_REJECTED: `bg-destructive/10 text-destructive`
- PUBLISHED: `bg-primary text-primary-foreground`

## Layout Contract

### Shell

```
┌─────────────────────────────────────────────────────┐
│ Topnav — h-14, border-b, bg-background, px-6        │
│ [Rosetta wordmark]              [reviewer name]      │
├─────────────────────────────────────────────────────┤
│ [Queue depth alert banner — full width, conditional] │
├─────────────────────────────────────────────────────┤
│ Queue page OR Split-pane review view                 │
└─────────────────────────────────────────────────────┘
```

### Queue Page (`/dashboard`)

```
┌──────────────────────────────────────────┐
│ Filter bar: [Search input] [Status ▼] [Severity ▼] [Assignee ▼]  │
├──────────────────────────────────────────┤
│ Queue item row (compact, single line):   │
│ [SEVERITY] Article Title / Section · N claims · [Assignee ▼] [STATUS] │
│ ...                                      │
└──────────────────────────────────────────┘
```

Queue items are sorted: HIGH severity → MED → LOW, then newest first within severity.
Each row is clickable — navigates to `/dashboard/review/[claimId]`.

Pagination: paginated (not infinite scroll) — 25 items per page. Rationale: predictable position for keyboard navigation; queue depth alert needs count visibility.

### Split Pane Review View (`/dashboard/review/[claimId]`)

```
┌─────────────────────┬─────────────────────┐
│  LEFT PANE (50%)    │  RIGHT PANE (50%)   │
│  Wikipedia para-    │  AI Commentary      │
│  graph with claims  │  + Sources checklist│
│  highlighted inline │  + Action buttons   │
│  (char offsets)     │  + Activity feed    │
│                     │                     │
│  Text selection     │  [Edit] [Approve]   │
│  → Flag button      │  [Reject] [Next]    │
└─────────────────────┴─────────────────────┘
```

Both panes use `ScrollArea` for independent scroll. Pane content is full viewport height minus topnav (calc(100vh - 56px)).

## Interaction Contract

### Queue Item Row

- Hover: `bg-muted` background transition (150ms ease)
- Selected/active: left border 2px `--primary`, `bg-muted` persistent
- Click: navigate to review view (no modal)
- Assignee dropdown: `Select` component, inline on the row, stops click propagation

### Source Verification (D-04, D-05, D-06)

- Each source displayed as a `Card` with: URL (truncated), title, relevance note
- Rating selector: three segmented buttons (Confirms / Partially supports / Does not support) with traffic light colors
- Approve button is `disabled` until all sources have a rating (enforced in component state AND API)
- Progress counter: "X of Y sources verified" — `text-sm text-muted-foreground`, counter text switches to `text-green-600` when X === Y

### Inline Commentary Editing (D-08)

- Edit button click: commentary `<p>` becomes `<Textarea>` in-place, no modal
- Save / Cancel buttons appear below textarea
- Unsaved changes show a dot indicator on the Edit button label
- On save: optimistic update, revert on API error with `Alert` toast

### Text Selection Flagging (D-09)

- Reviewer selects text in left pane
- A floating "Flag" button appears at the selection end point (position: absolute, z-index above content)
- Click Flag: inline mini form slides down below the paragraph (not modal): severity select + notes textarea + [Submit Flag] button
- On submit: new claim appears in right pane immediately (optimistic), paragraph highlight updates

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `a` | Approve (when all sources verified) |
| `r` | Reject |
| `e` | Focus commentary edit textarea |
| `j` / `k` | Next / previous queue item |
| `Escape` | Cancel edit / dismiss flag form |

Shortcut hints shown as `Tooltip` on hover of each action button.

### Loading States

- Queue page initial load: `Skeleton` rows (3 visible) matching queue item row height
- Review pane load: `Skeleton` blocks for left content and right commentary
- Action button during submission: spinner inside button, button `disabled`

### Queue Depth Alert (D-13)

- When unreviewed item count > configurable threshold (default: 50):
- Full-width `Alert` banner below topnav, `variant="destructive"`
- Copy: "Review queue has [N] unreviewed items — above the [threshold] item threshold."
- Banner is dismissible per session (localStorage key, not server state)

## Accessibility Contract

- All interactive elements reachable via keyboard Tab order
- Approve button `aria-disabled` (not `disabled`) when sources not yet verified, so screen readers announce the blocked reason via `aria-describedby` pointing to the "Verify all sources" tooltip
- Severity badges use both color and text label — never color alone
- Status badges use both color and text label — never color alone
- Traffic light source rating buttons use `aria-label`: "Mark as: Confirms", "Mark as: Partially supports", "Mark as: Does not support"
- Text selection flag button announces via `aria-live="polite"` when flag form opens
- Activity feed items use `<time datetime="...">` for timestamps
